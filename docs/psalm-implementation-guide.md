### 2. Text Completeness

**Issue:** Current implementation includes only opening verses (usually 1-3 verses). Full psalms need to be included.

**Impact:** 
- Script warnings: "Text is short" and "Text ends without punctuation"
- Presentation: Congregations typically sing entire psalms for meaningful context

**Warnings generated by abc_to_musiqwik.py:**
- All 6 completed psalms: "Text ends without sentence-final punctuation — may be truncated"
- Psalms 12, 18, 25, 117, 118: "Canticle text is short (32-48 words) — Verify all scriptural verses are included"

**Resolution:** Include all verses of each psalm from WEB-UE translation.

### 3. Gloria Patri (Doxology) Not Included

**Issue:** Gregorian chant psalms traditionally append Gloria Patri ("Glory be to the Father...") after the final verse.

**Current state:** Not included in generated files.

**Lutheran practice:** 
- LSB (Lutheran Service Book) includes Gloria Patri after each canticle
- Office settings typically append doxology to psalms
- Allows congregation to respond with "Amen"

**Resolution options:**
1. **Include Gloria Patri:** Add "Glory be to the Father, and to the Son, and to the Holy Spirit; as it was in the beginning, is now, and ever shall be, world without end. Amen."
2. **Mark as optional:** Add note in file that doxology may be appended as local practice dictates
3. **Create separate setting:** 8.XXX.2 for psalms with doxology, 8.XXX.1 for psalms without

**Recommendation:** Include doxology by default for Lutheran use; users can omit for other traditions.

### 4. Antiphon Format Clarification

**Issue:** Current implementation uses simple inline markers `[Antiphon]` and `[Antiphon closes]`.

**Current format:**
```
[Antiphon] Blessed is the man [Psalm text follows]

[full text]

[Antiphon closes] Blessed is the man
```

**Liturgical practice:** Antiphons typically:
- Open the psalm (full antiphon sung)
- Close the psalm (full antiphon repeated)
- May include responsorial elements in congregational settings

**Resolution:** Document antiphon structure and consider expanding format for responsorial use.

## Automation Script

### Usage

The `psalm_generator.py` script automates the creation of psalm files with proper structure.

#### Generate single psalm:
```bash
python3 psalm_generator.py --psalm 1 --tone 1 --force
```

#### Generate batch (all 6 completed psalms):
```bash
python3 psalm_generator.py --batch 1,12,18,25,117,118 --force
```

#### Generate all remaining psalms (sequential):
```bash
python3 psalm_generator.py --batch 2-150 --force
```

#### With custom text file (JSON):
```bash
python3 psalm_generator.py --batch 1-150 --text-file psalms_texts.json --force
```

### Script Features

- **Tone assignment:** Auto-assigns from `ANTIPHON_ASSIGNMENTS` dict
- **Text sourcing:** Reads from `PSALM_TEXTS` dict or external JSON file
- **File creation:** Generates properly formatted files with ABC notation
- **Validation:** Checks for existing files; warns before overwriting

### Text File Format (JSON)

For batch updates with full WEB-UE texts:

```json
{
  "1": "Full text of Psalm 1 verses 1-N...",
  "2": "Full text of Psalm 2 verses 1-N...",
  ...
  "150": "Full text of Psalm 150 verses 1-N..."
}
```

## Known Limitations

1. **Environment restrictions:** eBible.org and similar sites return 403 Forbidden for automated access
2. **Text sourcing:** Must use alternative methods or offline resources for WEB-UE texts
3. **No automatic antiphon generation:** Antiphons must be manually defined (currently 6 psalms only)
4. **No alternate settings:** Current implementation is single-tone only; multiple settings (8.XXX.1, .2, etc.) require separate runs
