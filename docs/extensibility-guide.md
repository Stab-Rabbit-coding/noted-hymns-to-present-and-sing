# Extensibility Guide — noted-hymns-to-present-and-sing

This guide explains how to contribute hymn files from traditions beyond the initial
Lutheran Service Book (LSB) scope, including Roman Catholic, Reformed, Baptist,
Anglican, Eastern Orthodox, and Charismatic/Pentecostal traditions.

## 1. File Format Reference

Every hymn file follows this exact layout:

```
<Full Hymn Title>

Tags: <comma-separated tags — see §2 below>

# Melody

## ABC
<ABC notation — see §5 below>

## Musiquik
<MusiQwik font string — generated by abc_to_musiqwik.py>

#Lyrics
<All verses, continuous prose. Verse numbers inline: "2.", "3.", "4.">

#Citations and References

Words: <Author, Year. Translation credit if applicable.>
Music: '<Tune name>' <Composer, Year.>
Setting: <Setting source, Year.>
copyright: <Status — public domain or license statement>

<Source URL>
```

### Section headers

| Header | Required | Notes |
|--------|----------|-------|
| `# Melody` | Yes | Contains `## ABC` and `## Musiquik` subsections |
| `#Lyrics` | Yes | All verses as continuous prose |
| `#Citations and References` | Yes | Full attribution, one field per line |

### Lyrics formatting

- All stanzas in a single prose block; no hard line breaks within a stanza.
- Verse 1 has no number. Subsequent stanzas begin with `2.`, `3.`, etc.
- Refrains, choruses, and responses are written **once** at the bottom of the
  `#Lyrics` block, after a blank line, labeled `Refrain:`, `Chorus:`, `Response:`,
  or `Antiphon:`. Each stanza references them inline with `[Refrain]` etc.
- Stanza-level tags use the syntax `[Tags: tag1, tag2]` placed immediately after
  the verse number (e.g. `2. [Tags: roman] Second verse text...`).

### File name

Plain text, no extension, underscores for spaces. Match the first distinctive words
of the English title:

```
A_Mighty_Fortress_Trusty_Shield
Beautiful_Savior
Now_Thank_We_All_Our_God
```

## 2. Theological Tag Taxonomy

Tags are comma-separated, lowercase, hyphenated identifiers on the `Tags:` line.
Apply every tag that accurately describes the piece.

### Tradition tags

Identify the liturgical or confessional tradition from which a piece originates.
The **file-level** tag reflects the broadest tradition covered by any stanza.

| Tag | Use |
|-----|-----|
| `lutheran` | Lutheran tradition (LCMS, ELCA, LMS, etc.); default for all LSB content |
| `roman` | Roman Catholic tradition (Tridentine and Novus Ordo) |
| `reformed` | Reformed / Calvinist (Presbyterian, Dutch Reformed, etc.) |
| `baptist` | Baptist tradition (Southern Baptist, Independent Baptist, etc.) |
| `anglican` | Anglican / Episcopal (BCP-based liturgy and hymnody) |
| `ecumenical` | Shared across multiple traditions without strong confessional identity |
| `eastern` | Eastern Orthodox (Greek, Russian, Coptic, Ethiopian, Armenian, etc.) |
| `charismatic` | Charismatic / Pentecostal (including neo-charismatic and renewal movements) |

### Theological content tags

Apply only when the doctrine is the **primary subject** of the text, not merely
mentioned in passing.

**Sacramental theology**

| Tag | Applies to |
|-----|-----------|
| `sacramental` | Any piece centred on the sacraments as means of grace |
| `real-presence` | Explicitly affirms Christ's body and blood in the Lord's Supper |
| `baptismal-regeneration` | Baptism as the means of new birth / regeneration |
| `absolution` | Confession, the Office of the Keys, and Holy Absolution |
| `lords-supper` | Lord's Supper / Holy Communion focus |
| `baptism` | Baptism focus (may or may not include regeneration language) |

**Reformation theology**

| Tag | Applies to |
|-----|-----------|
| `reformation` | Broadly Reformation in character (Luther, Calvin, Zwingli era) |
| `sola-scriptura` | Scripture alone as the source and norm of doctrine |
| `sola-fide` | Faith alone as the means of justification |
| `sola-gratia` | Grace alone as the ground of salvation |
| `solus-christus` | Christ alone as mediator and redeemer |

**Cross-tradition markers**

| Tag | Applies to |
|-----|-----------|
| `saints` | Invocation of saints or angels as intercessors (Roman, Eastern, Anglican; **contra** Lutheran, Reformed, Baptist, Charismatic) |
| `decision` | Decision theology — regeneration contingent on a human act of will / altar call (Baptist, revivalist; **contra** Lutheran, Roman, Eastern, Reformed) |
| `gifts` | Active continuationist spiritual gifts — tongues, prophecy, healing (Charismatic; **contra** Baptist, Reformed) |

### Form and function tags

| Tag | Applies to |
|-----|-----------|
| `liturgical` | Part of a formal liturgical order (Divine Service, Daily Office) |
| `office` | Daily Office (Matins, Vespers, Compline, etc.) |
| `canticle` | Prose canticle chanted to a psalm tone |
| `psalm` | Psalm setting (metrical or chanted) |
| `creed` | Credal text (Nicene, Apostles', Athanasian) |
| `intercession` | Primarily a prayer of intercession for others |

### Incompatibility rules

The `abc_to_musiqwik.py` script warns when a tag set combines a tradition tag with
a theological tag that is incompatible with that tradition:

| Tradition | Incompatible theological tags |
|-----------|-------------------------------|
| `lutheran` | `saints`, `decision` |
| `reformed` | `saints`, `real-presence`, `baptismal-regeneration`, `decision`, `gifts` |
| `baptist` | `saints`, `sacramental`, `real-presence`, `baptismal-regeneration`, `absolution`, `lords-supper`, `gifts` |
| `roman` | `sola-scriptura`, `sola-fide`, `sola-gratia`, `solus-christus`, `decision`, `reformation` |
| `anglican` | `decision` |
| `eastern` | `sola-scriptura`, `sola-fide`, `sola-gratia`, `solus-christus`, `decision`, `reformation` |
| `charismatic` | `saints` |

These are warnings, not errors. A warning means the tag combination is unusual and
should be reviewed; it does not prevent the file from being used.

### Examples

```
# Roman Rite canticle
Tags: roman, liturgical, canticle

# A psalm setting usable across traditions
Tags: lutheran, reformed, anglican, ecumenical, psalm

# Charismatic praise song with continuationist emphasis
Tags: charismatic, gifts

# Baptismal hymn adopted across Western traditions
Tags: lutheran, roman, ecumenical, sacramental, baptism, baptismal-regeneration
```

## 3. Folder Naming Convention

Hymn files live under `hymns/<section_number>_<Section_Name>/`.

```
hymns/
├── 2.0_Divine_Service_Liturgy/
├── 3.0_Matins/
├── 7.5_Holy_Week/
│   ├── O_Sacred_Head_Now_Wounded
│   └── Not_All_the_Blood_of_Beasts
└── <new section dirs…>/
```

### Rules

- Directory names begin with the two-digit TODO section number so `ls` sorts them
  numerically.
- Underscores replace spaces in both directory and file names.
- No file extensions.
- File names match the first distinctive words of the English title.
- For new sections outside the existing numbering scheme (e.g. a Roman Rite
  section), contact the maintainers to assign a section number, or use a
  contributor-prefixed placeholder such as `contrib/<YourName>/<filename>` while
  the section is being formally incorporated.

### Adding a new section

1. Decide the section number and descriptive name in consultation with the
   maintainers (file a GitHub issue if needed).
2. Create the directory: `hymns/<NN>_<Section_Name>/`
3. Add the section heading and item list to `TODO.md` following the existing
   format.
4. Begin adding hymn files.

## 4. Copyright Verification Checklist

**All three components — words, music, and setting — must be independently
verified as public domain or explicitly licensed for free reproduction without
royalties.** A piece whose words are public domain is still excluded if the
setting (arrangement, harmonization) is under copyright.

### Public domain threshold (United States)

Works first published before 1 January 1928 are in the public domain in the US
regardless of the author's death date. For works published from 1928 onward,
copyright generally lasts 70 years after the author's death (or 95 years from
publication for corporate works), subject to renewal rules for works published
1928–1963.

### Pre-flight checklist

Before creating a hymn file, confirm each item:

- [ ] **Words** — Who wrote the original text? What year?
  - If a translation: who translated? What year? Is the translation itself
    pre-1928 or by an author who died before 1955?
- [ ] **Music** — Who composed the tune? What year?
  - If a harmonization: who harmonized? What year?
- [ ] **Setting** — When was the specific arrangement first published?
  - A 20th-century choral setting of an 18th-century tune may be under copyright
    even if the tune is not.
- [ ] **Source verification** — Can you point to a primary source (original
    publication, hymnary.org, Open Hymnal Project, IMSLP, etc.) that confirms the
    dates above?
- [ ] **No known conflicts** — Has any rights holder claimed copyright in this
    specific text/tune/setting combination in the past 10 years?

### Reliable public-domain sources

| Source | What it covers |
|--------|----------------|
| Open Hymnal Project (openhymnal.org) | Lutheran-tradition ABC files, copyright-verified |
| Hymnary.org | Bibliographic data and copyright status notes |
| IMSLP (imslp.org) | Music scores; PD determinations per country |
| CCEL (ccel.org) | Historic Christian texts and hymn collections |
| Project Gutenberg (gutenberg.org) | Pre-1928 US texts |
| British Library digitized collections | Pre-1928 UK texts |

### Special cases

**Translations** — Even if the original is ancient, a 20th-century translation
may still be under copyright. The English translation must also be pre-1928 or
explicitly licensed. Example: a new translation of Luther's German published
in 1952 is likely still under copyright.

**Modern harmonizations** — A new SATB arrangement (even of an 18th-century tune)
may be under copyright. Use only harmonizations first published before 1928, or
use unharmonized melody lines.

**LSB copyright items** — The Lutheran Service Book (2006) contains both public
domain and copyrighted material. Do not copy text or music directly from the LSB
without independent verification that the underlying source is public domain.
Cross-reference with the Open Hymnal Project and hymnary.org.

**[!] markers** — Items in `TODO.md` marked `[!]` require copyright verification
before work begins. Do not create files for these items until the copyright status
is confirmed.

### When in doubt

When copyright status is uncertain:
1. Do not include the piece.
2. File a GitHub issue with the hymn title and your research so far.
3. A maintainer will investigate and either confirm public domain status or mark
   the item with `[!]`.

## 5. ABC Melody Encoding

Store the melody as ABC notation in the `## ABC` subsection under `# Melody`.

### Required header fields

```
X:1                  — tune index (always 1 per file)
T:Title              — tune title
C:Composer, Year     — composer and date
S:Source             — source attribution and URL
M:4/4                — meter
L:1/4                — default note length
K:D                  — key signature
```

### Pitch notation

- Uppercase `C D E F G A B` = middle octave (C4–B4)
- Lowercase `c d e f g a b` = octave above (C5–B5)
- Comma suffix = octave below: `C,` = C3
- Apostrophe suffix = octave above: `c'` = C6
- `^F` = F#, `_B` = Bb, `=B` = B natural

### Duration notation

- Bare letter = default length (L: field)
- `D2` = double, `D4` = quadruple, `D/` = half default
- `D3/2` = dotted (1.5× default)

### MusiQwik range constraint

The MusiQwik font covers only A3–A5 (MIDI 57–81). When writing ABC:
- Avoid notes below A3 or above A5.
- If the melody sits too high or low, transpose the key.
- The `abc_to_musiqwik.py` script warns about out-of-range notes and skips them;
  add these notes manually in the presentation text box.

### Running the converter

```
python3 abc_to_musiqwik.py --file hymns/<section_dir>/<Hymn_File_Name>
```

The script writes the MusiQwik string directly into the `## Musiquik` section of
the same file. Run it every time the `## ABC` subsection changes.

## 6. Adding a Hymn — Step-by-Step

1. **Verify copyright** — complete the checklist in §4 above.
2. **Find the ABC source** — check the Open Hymnal Project
   (`openhymnal.org/Abc/<filename>.abc`) or write the ABC by hand from a
   public-domain score.
3. **Create the file** — under `hymns/<section_dir>/` using the naming convention
   from §3.
4. **Populate `## ABC`** — paste or write the verified ABC notation.
5. **Run the converter** — `python3 abc_to_musiqwik.py --file <path>` — confirm
   no unexpected warnings. Dotted-note approximations are expected; out-of-range
   note warnings require transposing the key.
6. **Populate `#Lyrics`** — all stanzas as continuous prose with inline verse
   numbers. Include `[Refrain]` / `Refrain:` structure if the hymn has a repeating
   segment.
7. **Populate `#Citations and References`** — words, music, setting, copyright
   statement, and source URL.
8. **Update `TODO.md`** — change `[ ]` to `[x]` and add the `[FILE: ...]` path.

## 7. Contributing from Non-Lutheran Traditions

The core collection covers Lutheran Service Book content. This section explains how
to add hymns from other traditions following the same conventions.

### Roman Catholic

- Use tag `roman` (and `ecumenical` if also used outside Roman tradition).
- Medieval Latin hymns (pre-Trent) are generally safe; post-Vatican II texts are
  often under copyright — verify before including.
- Gregorian chant notation: use ABC with meter `M:C` or `M:4/4`; note values
  approximate the chant syllables. Chant tones (psalm tones 1–8) are in the
  public domain.
- Roman Catholic theological tags that differ from Lutheran: `saints` and
  `real-presence` are appropriate for Roman theology. Do not apply `sola-fide`,
  `sola-gratia`, `sola-scriptura`, or `solus-christus` to Roman Catholic texts
  that do not explicitly teach these doctrines.

### Reformed / Presbyterian

- Use tag `reformed`.
- The Scottish Psalter (1650) is a major public-domain source; Watts (pre-1928
  translations), the Bay Psalm Book (1640), and the Geneva Psalter (1562) are
  also public domain.
- Reformed theology: apply `sola-scriptura`, `sola-fide`, `sola-gratia`,
  `solus-christus` as appropriate. Do not apply `real-presence`, `baptismal-
  regeneration`, or `lords-supper` unless explicitly warranted by the text.

### Anglican / Episcopal

- Use tag `anglican` (and `ecumenical` if applicable).
- The 1662 Book of Common Prayer texts are public domain. Hymns from the English
  Hymnal (1906) and Hymns Ancient and Modern (pre-1928 editions) are generally
  safe — verify the specific edition.
- Anglican theological tags: `saints` may apply where the text invokes saints
  as intercessors; `real-presence` for sacramental texts in the high-church
  tradition; avoid `decision` (not characteristic of Anglican theology).

### Baptist

- Use tag `baptist`.
- The Baptist tradition has a strong hymnody in the 18th–19th centuries: Watts,
  Cowper, Newton, Toplady. Most pre-1928 texts are public domain.
- Baptist theological tags: `decision` may apply where the text reflects
  revivalist/altar-call theology. Do not apply `sacramental`, `real-presence`,
  `baptismal-regeneration`, or `lords-supper` — Baptist theology treats Baptism
  and the Lord's Supper as ordinances rather than sacraments.
- Baptist hymns often appear in the Leavitt and Mason collections (early 19th c.),
  the Southern Baptist Hymnal (1940 ed. is in the PD threshold), and the Silas
  Jones Sacred Harp tradition (1835).

### Eastern Orthodox

- Use tag `eastern`.
- Byzantine liturgical texts (pre-schism and early Eastern church) are generally
  pre-copyright; modern English translations may not be.
- Eastern Orthodox theological tags: `saints` is appropriate (significant in
  Eastern spirituality); `liturgical` for Divine Liturgy content. Do not apply
  Reformation-era tags (`sola-scriptura`, `sola-fide`, etc.) to Eastern texts.
- ABC notation for Byzantine chant: use unmetered notation `M:none` or
  approximate in `M:4/4`; note the approximation in the citations.

### Charismatic / Pentecostal

- Use tag `charismatic`.
- Most Charismatic praise music is 20th-century and under copyright. The
  exception is earlier Gospel-era songs (pre-1928). Verify carefully.
- Charismatic theological tags: `gifts` for texts emphasizing active spiritual
  gifts; `decision` for altar-call texts. Do not apply `saints`.

### Stanza-level tags for mixed-tradition hymns

When a hymn originates in one tradition but individual stanzas were added by or
are exclusively used in another, tag those stanzas separately:

```
#Lyrics
First verse text...  2. [Tags: anglican] Verse added for Anglican naval use...
```

The file-level `Tags:` line should cover the **broadest** tradition present in
any stanza. Stanza tags narrow that down.

## Quick Reference: Tag Decision Tree

```
Is the piece from the Lutheran tradition?
  Yes → include 'lutheran'
  No  → include the relevant tradition tag(s)

Does the piece invoke saints or angels as intercessors?
  Yes → add 'saints'; ensure 'lutheran', 'reformed', 'baptist', 'charismatic'
        are NOT on the same stanza (or use stanza-level tags)

Does the text explicitly teach baptismal regeneration?
  Yes → add 'baptismal-regeneration' and 'baptism' and 'sacramental'

Does the text teach decision theology / altar call?
  Yes → add 'decision'; ensure 'lutheran', 'roman', 'eastern', 'reformed'
        are NOT on the same stanza

Does the text emphasize continuing spiritual gifts (tongues, prophecy, healing)?
  Yes → add 'gifts'

Is the piece part of a formal liturgical rite?
  Yes → add 'liturgical'; if Daily Office, also add 'office'
```
